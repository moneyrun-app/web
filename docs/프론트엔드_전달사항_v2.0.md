## 1. 전체 변경 요약

### 삭제해야 할 것
- 트랙타일 전체 (매일 소비체크, 주간 요약, 월간 확정/취소 UI)
- 월간 리포트 페이지
- 학습 콘텐츠 페이지 (learn)
- 학습 스크랩 기능 (user_content_scraps)
- 기존 퀴즈 10개 표시 → 1개로 변경

### 새로 만들어야 할 것
- 머니북 (서점) 페이지
- 마이북 (내 서재) 머니레터 페이지
- 책 열람 뷰 + 하이라이트(책갈피) 기능
- 스크랩 모아보기 (외부 URL + 퀴즈 스크랩 통합)
- 퀴즈 스크랩(북마크) 기능
- 출석체크 UI (스트릭 표시, 캘린더)
- 퀴즈 난이도 조절 UI
- 온보딩에 월 투자액 필드 추가
- 머니북 구매 시 추가 온보딩 폼 (동적 필드)

### 수정해야 할 것
- 하단 탭 네비게이션: 3탭 → 4탭
- 페이스메이커 화면: 퀴즈 1개 + 출석 표시 + AI 메시지
- 마이페이지: 퀴즈 난이도/출석/뱃지 추가

```typescript
// Request
{
```

### 3-3. 신규 API

#### 퀴즈/출석 API

**`POST /quiz/:id/answer`** — 퀴즈 답변 제출 (출석 자동 체크)
```typescript

// Request
{ note?: string; }

// Response
{ scrapped: boolean; scrapId: string | null; }
```

```typescript
// Request

// Response
{ newLevel: number; }
```

**`GET /attendance/streak`** — 출석 현황 + 뱃지

```typescript
// Response

    code: string; name: string; icon: string; earnedAt: string;
```

**`GET /attendance/history`** — 출석 캘린더 (월별)

```typescript
// Query: ?month=2026-04

}
```

## 4. 페이지별 개발 가이드

### 4-1. 온보딩 — 월 투자액 필드 추가

기존 7개 필드 → 8개 필드. `monthlyInvestment` 필드를 월 실수령액 다음, 월 고정비 이전에 추가.

### 4-2. 페이스메이커 (홈)

- 출석 스트릭 바 (attendance.currentStreak + 이번 주 출석 시각화)
- 오늘의 퀴즈 1개 (todayQuiz) — null이면 "완료" 표시
- 퀴즈 제출 → 정답/오답 모달 + 스크랩 버튼 + 난이도 변경 제안
- AI 메시지 + 명언 + 면책문구

### 4-3. 머니북 (서점)

- 카테고리 필터 탭 + 책 카드 그리드
- 책 상세: 커버 + 챕터 미리보기 + 동적 폼(requiredFields) + 구매 플로우

### 4-4. 마이북 (내 서재)

- 머니레터 메인: 상세리포트 + 구매한 책 + 바로가기
- 책 열람: 챕터 탭 + 마크다운 렌더링 + 하이라이트 기능
- 하이라이트 모아보기: 색상 필터
- 스크랩 모아보기: URL/퀴즈 통합 + 스크랩 책 생성

### 4-5. 마이페이지 — 퀴즈 난이도/출석/뱃지 추가

## 6. 어드민 머니북 관리

### 6-1. 화면 구성

- **탭 위치:** 어드민 네비 — 유저 | 퀴즈 | **머니북** | 설정
- **메인 뷰:** 등록된 책 목록 (총 권수 표시) + "새 책 등록" 버튼
- **목록 카드:** 카테고리 뱃지 + 제목 + 챕터수/구매수 — 펼치면 상세 + 액션 버튼
- **액션:** 챕터 관리 / 수정 / 삭제

### 6-2. API 스펙

**책 CRUD:**
```
GET    /admin/money-book             → { books: AdminBook[], total: number }
GET    /admin/money-book/:id         → { book: AdminBook, chapters: AdminBookChapter[] }
POST   /admin/money-book             → AdminBook
PUT    /admin/money-book/:id         → AdminBook
DELETE /admin/money-book/:id         → { success: true }
```

**챕터 CRUD:**
```
PUT    /admin/money-book/:id/chapters → { chapters: AdminBookChapter[] }
```

**타입:**
```typescript
interface AdminBook {
  id: string;
  title: string;
  description: string;
  category: string;       // tax | pension | realestate | stock | insurance | savings
  coverImageUrl: string | null;
  chapterCount: number;
  purchaseCount: number;
  requiredFields: AdminRequiredField[];
  createdAt: string;
}

interface AdminRequiredField {
  key: string;
  label: string;
  type: 'number' | 'text' | 'select';
  options?: string[];
  placeholder?: string;
  required: boolean;
}

interface AdminBookChapter {
  id: string;
  order: number;
  title: string;
  promptTemplate: string;  // AI 생성용 프롬프트 템플릿
}
```

### 6-3. 프롬프트 템플릿 작성 가이드

챕터의 `promptTemplate`에서 사용 가능한 변수:

| 변수 | 설명 | 예시 |
|---|---|---|
| `{{nickname}}` | 유저 닉네임 | 시은 |
| `{{age}}` | 나이 | 28 |
| `{{monthlyIncome}}` | 월 소득 | 3000000 |
| `{{monthlyInvestment}}` | 월 투자액 | 500000 |
| `{{grade}}` | 등급 | RED / YELLOW / GREEN |
| `{{extra.KEY}}` | 추가 온보딩 필드 값 | `{{extra.annual_income}}` |

**프롬프트 예시 (연말정산 Ch.1):**
```
{{nickname}}님은 연소득 {{extra.annual_income}}원이고
부양가족 {{extra.dependents}}명인 직장인입니다.
주거형태는 {{extra.housing_type}}이며,
카드 사용액은 {{extra.card_spending}}원입니다.

이 정보를 바탕으로 2026년 연말정산 절세 전략을 설명해주세요.
소득공제, 세액공제 항목별로 나눠서 작성하세요.
```

### 6-4. 예시 책 2권 (시드 데이터)

**1. 2026 연말정산 완전정복**
- 카테고리: `tax`
- 5챕터: 소득공제 총정리 / 세액공제 핵심 / 의료비·교육비 / 카드 전략 / 실전 체크리스트
- 추가 온보딩 필드:
  - `annual_income` (숫자) — 연소득
  - `dependents` (숫자) — 부양가족 수
  - `housing_type` (선택: 월세, 전세, 자가) — 주거형태
  - `card_spending` (숫자) — 연간 카드사용액

**2. 퇴직연금, 이것만 알면 된다**
- 카테고리: `pension`
- 5챕터: DC vs DB 비교 / IRP 활용법 / 세액공제 극대화 / 투자전략 / 수령 시 절세
- 추가 온보딩 필드:
  - `pension_type` (선택: DC, DB, IRP, 모름) — 연금유형
  - `pension_balance` (숫자) — 적립금
  - `years_worked` (숫자) — 근속연수
  - `extra_contribution` (숫자) — 추가납입액

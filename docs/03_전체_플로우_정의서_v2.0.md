# 머니런 전체 플로우 정의서

> **버전:** v3.0 — 현행 코드 기준 전면 업데이트
> **최종 수정:** 2026.04.08

---

## 1. 유저 여정 전체 맵

```
[비로그인 유저]
    |
    v
  랜딩 (/) — 시뮬레이션 위저드
  닉네임 → 나이 → 은퇴나이 → 연금나이 → 실수령액 → 고정비 → 변동비
  → 시뮬레이션 결과 (현재/미래/CTA)
  → "카카오로 시작하기"
    |  카카오 로그인
    v
  앱 레이아웃 초기화 (/(app)/layout.tsx)
  ├── 신규 유저: 시뮬레이션 데이터로 자동 온보딩
  └── 기존 유저: 프로필 로드 + 프로필 충돌 시 업데이트 확인
    |
    v
  페이스메이커 (/home) ← 매일 허브
  AI 메시지 + 소비 트랙 + 월간 확정 + 퀴즈
    |                    |                    |
    v                    v                    v
  마이북 (/book)    마이 (/my)          어드민 (/admin/*)
  리포트/오답/스크랩  프로필 수정          유저/퀴즈/설정 관리
```

## 2. 인증 + 온보딩 플로우

```
[프론트]                    [NextAuth]              [NestJS]
   |                           |                       |
   | "카카오로 시작하기"       |                       |
   |-------------------------->|                       |
   |                   카카오 OAuth                    |
   |<-- kakaoAccessToken ------|                       |
   |                                                   |
   | POST /auth/kakao { accessToken } --------------->|
   |<-- { accessToken: JWT, user: { isNewUser } } ----|
   |                                                   |
   | JWT를 sessionStorage에 저장                       |
   |                                                   |
   | isNewUser === true?                               |
   | YES → 세션스토리지에서 시뮬레이션 데이터 로드      |
   |   POST /auth/onboarding { nickname, age,          |
   |     retirementAge, pensionStartAge,                |
   |     monthlyIncome, monthlyFixedCost,               |
   |     monthlyVariableCost, goodSpendings,             |
   |     fixedExpenses } ------------------------------>|
   |<-- { grade, surplus } ----------------------------|
   |   시뮬레이션 세션 데이터 삭제                      |
   |   → /home                                         |
   |                                                   |
   | NO (기존 유저)                                    |
   |   GET /finance/profile ----->                     |
   |   GET /users/me ------------->                    |
   |   시뮬레이션 데이터 vs DB 프로필 비교             |
   |   다르면 → 업데이트 다이얼로그                    |
   |   → /home                                         |
```

## 3. 페이스메이커 — 일일 사이클

```
[유저가 /home 진입]
    |
    v
GET /pacemaker/today
    |
    |-- [AI 메시지 표시]
    |   |-- 오늘의 한마디 (테마 태그 포함)
    |   +-- 추천 행동 → 마이북 콘텐츠 링크
    |
    |-- [월간 소비 트랙]
    |   GET /pacemaker/daily-checks?month=2026-04
    |   |-- 캘린더 그리드 표시 (일별 초록/노랑/빨강)
    |   |-- 날짜 클릭 → 데일리 체크 모달
    |   |   POST /pacemaker/daily-check { date, status, amount }
    |   +-- 좌우 스와이프 → 월 전환
    |
    |-- [데일리 퀴즈]
    |   |-- 객관식 4지선다 표시
    |   |-- POST /pacemaker/quiz/:quizId/answer { userAnswer }
    |   |-- 정답: 설명 표시 (간단 + 상세)
    |   +-- 오답: wrong_notes에 자동 기록
    |
    +-- [월간 확정]
        GET /pacemaker/monthly-finalize-status
        |-- 확정 가능 → POST /pacemaker/monthly-finalize { month }
        |-- 확정 취소 → POST /pacemaker/monthly-finalize/cancel { month }
        +-- 미확정 과거 월 알림
```

## 4. 마이북 데이터 흐름

```
마이북 (/book) — 3탭
  |
  |-- [리포트] 탭
  |   |-- 상세 리포트 영역
  |   |   GET /book/detailed-reports → 목록
  |   |   클릭 → /book/{id}?type=detailed → GET /book/detailed-reports/:id
  |   |   ├── V6: sections 배열 → 26개 섹션 컴포넌트로 렌더링
  |   |   └── 레거시: 마크다운 문자열 파싱
  |   |
  |   +-- 월간 리포트 영역 (구분선 아래)
  |       GET /book/monthly-reports → 목록 (pending/created 구분)
  |       pending → "리포트 만들기" 버튼
  |         1단계: 체감 선택 (good/okay/tight/bad)
  |         2단계: 제안 항목 체크 (GET /book/monthly-reports/proposals)
  |         3단계: 메모 입력
  |         → POST /book/monthly-reports
  |       created → 클릭 → /book/{id}?type=monthly → GET /book/monthly-reports/:id
  |
  |-- [오답노트] 탭
  |   GET /book/wrong-notes → 목록
  |   |-- 내 답 (빨강 배지) + 정답 (초록 배지)
  |   |-- 한줄 설명
  |   +-- 펼침 → 상세 설명 (마크다운)
  |
  +-- [스크랩] 탭
      GET /book/scraps → 목록
      |-- 외부 URL 스크랩 (채널, 작성자, AI 요약)
      |-- FAB 버튼 → URL 입력 모달
      |   POST /book/scraps { url }
      +-- 삭제 → DELETE /book/scraps/:id
```

```
[앱 시작]

    |   |-- sessionStorage에서 JWT 복원 시도
    |   |-- 없으면 NextAuth → 카카오 토큰 → POST /auth/kakao → JWT
    |   |-- GET /finance/profile → financeStore
    |   |-- GET /users/me → userStore
    |   +-- GET /constants → constantsStore (useConstants 훅, 1시간 캐시)
    |
    +-- /home 진입 시:
        +-- GET /pacemaker/today → 오늘의 메시지 + 퀴즈
```

## 7. 상태 관리 맵

```
Zustand Stores
  |-- userStore        → id, nickname, email, role, hasCompletedOnboarding, isLoggedIn
  |-- financeStore     → nickname, age, retirementAge, pensionStartAge,
  |                       monthlyIncome, monthlyFixedCost, monthlyVariableCost,
  |                       grade, surplus, variableCost { daily, weekly, monthly }
  |-- simulationStore  → 시뮬레이션 입력값 (sessionStorage 영속화)
  |-- constantsStore   → seoulAverageRent, categoryAverages, inflationRate
  +-- themeStore       → theme (light/dark), toggleTheme(), initTheme()

TanStack Query Keys
  |-- ['constants']                    → GET /constants
  |-- ['pacemaker-today']             → GET /pacemaker/today
  |-- ['daily-checks', month]         → GET /pacemaker/daily-checks
  |-- ['weekly-summary', date]        → GET /pacemaker/weekly-summary
  |-- ['monthly-finalize-status']     → GET /pacemaker/monthly-finalize-status
  |-- ['finance-profile']             → GET /finance/profile
  |-- ['detailed-reports']            → GET /book/detailed-reports
  |-- ['detailed-report', id]         → GET /book/detailed-reports/:id
  |-- ['monthly-reports']             → GET /book/monthly-reports
  |-- ['monthly-report', id]          → GET /book/monthly-reports/:id
  |-- ['monthly-report-proposals']    → GET /book/monthly-reports/proposals
  |-- ['wrong-notes']                 → GET /book/wrong-notes
  |-- ['scraps']                      → GET /book/scraps
  |-- ['learn-contents', grade?]      → GET /book/learn
  |-- ['learn-content', id]           → GET /book/learn/:id
  |-- ['peer-statistics', age, income] → GET /statistics/peers
  |-- ['admin-users', page, limit]    → GET /admin/users
  +-- ['admin-quizzes']               → GET /admin/quizzes
```

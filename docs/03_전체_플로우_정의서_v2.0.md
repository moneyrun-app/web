# 머니런 전체 플로우 정의서

> **버전:** v2.0 — 3페이지 MVP

---

## 1. 유저 여정 전체 맵

```
[신규 유저]
    |
    v
  랜딩/로그인 (/)
  "카카오로 시작하기"
    |  카카오 로그인
    v
  온보딩 (/onboarding) ← 최초 1회
  나이 → 월 실수령액 → 좋은 소비 → 고정 소비
  → "하루 OO원"
    |
    v
  페이스메이커 (/home) ← 매일 허브
  AI 오늘의 한마디 + 하루 사용 가능
  추천 행동 → 마이북 콘텐츠 링크
    |                    |
    v                    v
  마이북 (/book)    마이 (/my)
  상세 리포트        정보 수정
  주간 리포트        → 잉여자금 재계산
  스크랩
  금융 학습
```

---

## 2. 인증 + 온보딩 플로우

```
[프론트]                    [NextAuth]              [NestJS]
   |                           |                       |
   | "카카오로 시작하기"       |                       |
   |-------------------------->|                       |
   |                   카카오 OAuth                    |
   |                           |-- POST /auth/kakao -->|
   |                           |<-- { jwt, isNewUser } |
   |<--------------------------|                       |
   |                                                   |
   | isNewUser === true?                               |
   | YES → /onboarding                                |
   |   유저가 4스텝 입력                                |
   |   POST /onboarding ------------------------------>|
   |     { age, monthlyIncome,                         | DB 저장
   |       goodSpendings, fixedExpenses }               | 잉여자금 계산
   |<-- { grade, surplus } ----------------------------|
   |   → /home                                         |
   |                                                   |
   | NO (hasCompletedOnboarding)                       |
   |     → /home                                       |
```

## 3. 일일 사이클 — 페이스메이커

```
[매일 아침, 유저가 앱을 연다]
    |
    v
GET /pacemaker/today
    |
    |-- 오늘 날짜로 pacemaker_messages 조회
    |
    |-- [이미 있음] → 캐시된 메시지 반환
    |
    +-- [없음] → AI 메시지 생성 시작
         |
         |-- 1. 참조 데이터 수집
         |   |-- 유저 재무 프로필 (나이, 실수령액, 잉여자금)
         |   |-- 등급 (RED/YELLOW/GREEN)
         |   |-- 최근 상세 리포트 (1건)
         |   |-- 최근 주간 리포트 (1건)
         |   |-- 유저 스크랩 목록 (최근 5건)
         |   |-- system_config (서울 평균 등)
         |   +-- 오늘 날짜/요일
         |
         |-- 2. Claude API 프롬프트 구성
         |-- 3. AI 응답 → 메시지 + 추천 행동(actions) 파싱
         |-- 4. pacemaker_messages에 저장
         +-- 5. 반환
```

## 4. 상세 리포트 — 프로필 변경 트리거

```
[유저가 마이페이지에서 재무 정보 수정]
    |
    |-- 잉여자금 재계산 (즉시)
    |
    |-- 마지막 상세 리포트 생성일 확인
    |   |-- 30일 미경과 → 생성 스킵
    |   |   → "다음 리포트는 OO일 후 가능" 안내
    |   |
    |   +-- 30일 경과 → AI 상세 리포트 생성
    |       |-- 유저 재무 프로필 + 잉여자금 + 등급
    |       |-- system_config (서울 평균 등)
    |       |-- Claude API 호출
    |       +-- detailed_reports에 저장
```

---

## 5. 주간 리포트 — 유저 입력 기반

```
[유저가 마이북 > 주간 리포트에서 "이번 주 리포트 만들기" 클릭]
    |
    |-- 이번 주 상태 입력 화면
    |   |-- 체감 선택: good / okay / tight / bad
    |   +-- 자유 메모 (선택)
    |
    |-- POST /book/weekly-reports
    |
    [백엔드]
    |   |-- 유저 입력 수신
    |   |-- 참조 데이터 수집
    |   |-- Claude API 호출 → 한 쪽짜리 가이드 생성
    |   +-- weekly_reports에 저장
```

## 6. 재무 정보 수정 → 재계산 플로우

```
[마이페이지에서 수정]

  [프론트]
    |-- Zustand financeStore 즉시 업데이트
    |   → 잉여자금 재계산 (lib/surplus.ts)
    |   → 등급 재판정 (lib/grade.ts)
    |   → 홈 탭 "하루 OO원" 즉시 반영
    |
    +-- PATCH /finance/profile

  [백엔드]
    |-- finance_profiles 업데이트
    |-- 잉여자금 재계산 + 등급 재판정
    +-- 응답: { grade, surplus }
```

## 7. 마이북 데이터 흐름

```
마이북 (/book)
  |
  |-- [상세 리포트] 탭
  |   +-- GET /book/detailed-reports → 목록
  |       +-- 클릭 → GET /book/detailed-reports/:id → 상세
  |
  |-- [주간 리포트] 탭
  |   +-- GET /book/weekly-reports → 목록
  |       +-- 클릭 → GET /book/weekly-reports/:id → 상세
  |
  |-- [스크랩] 탭
  |   +-- GET /book/scraps → 스크랩 목록
  |
  +-- [금융 학습] 탭
      +-- GET /book/learn → 등급별 목록
          +-- 클릭 → GET /book/learn/:id → 마크다운 본문
              +-- 스크랩 → POST /book/learn/:id/scrap
```

## 8. 앱 초기화 플로우

```
[앱 시작]
    |
    |-- GET /constants → constantsStore (1회)
    |
    |-- NextAuth 세션 확인
    |   |-- 없음 → 로그인 화면 (/)
    |   +-- 있음 → JWT 유효?
    |       |-- 유효 + 온보딩 완료 → /home
    |       |-- 유효 + 온보딩 미완료 → /onboarding
    |       +-- 만료 → 세션 갱신 → 실패 시 로그인으로
    |
    +-- /home 진입 시:
        |-- GET /finance/profile → financeStore
        +-- GET /pacemaker/today → 오늘의 메시지
```

## 9. 상태 관리 맵

```
Zustand Stores
  |-- userStore       → id, nickname, email, hasCompletedOnboarding
  |-- financeStore    → age, monthlyIncome, goodSpendings, fixedExpenses,
  |                      grade, surplus { monthly, weekly, daily }
  +-- constantsStore  → seoulAverageRent, categoryAverages, inflationRate

TanStack Query
  |-- ['pacemaker-today']      → GET /pacemaker/today
  |-- ['detailed-reports']     → GET /book/detailed-reports
  |-- ['weekly-reports']       → GET /book/weekly-reports
  |-- ['learn-contents']       → GET /book/learn
  +-- ['scraps']               → GET /book/scraps
```

# 프론트엔드 전달사항 — 머니런 v3.0 코스 시스템

> **작성일:** 2026-04-15
> **대상:** 프론트엔드 개발
> **상태:** 기획 확정 (백엔드 API 확정 후 작업 시작)

---

## 1. 전체 변경 요약

### 새로 만들어야 할 것
- **온보딩 5단계 UI** (관심 분야 → 진단퀴즈 → 데이터 입력 → 생성 로딩 → 웰컴)
- **온보딩 이어하기** (재접속 시 마지막 단계부터 재개)
- **코스 진도 표시** (페이스메이커 홈 + 마이페이지)
- **미션 UI** (미션 목록, 완료 체크)
- **코스 완료 → 다음 코스 선택 화면**

### 수정해야 할 것
- **페이스메이커 홈**: 코스 진도 바 추가, 퀴즈가 코스 범위
- **마이북 메인**: 코스 마이북 섹션 추가 (source: 'course')
- **마이페이지**: 현재 코스 정보 표시

### 유지 (변경 없음)
- 머니북(서점) 전체
- 하이라이트 시스템
- 스크랩 모아보기
- 오답 노트
- 출석/뱃지 시스템
- 어드민 페이지

### 3-1. 신규 API — 온보딩 v3

---

**`GET /course/onboarding/status`** 🔒 — 온보딩 진행 상태 조회 (이어하기용)

```typescript
// Response
{
  currentStep: number;           // 1~5 (어디까지 진행했는지)
  isComplete: boolean;           // 전체 완료 여부
  step1: {
    selectedCategory: string | null;  // '연금' | '주식' | '부동산' | '세금_연말정산' | '소비_저축'
  };
  step2: {
    assignedLevel: string | null;     // '기초' | '심화' | '마스터'
    diagnosticScore: number | null;   // 0~100
  };
  step3: {
    financeDataSubmitted: boolean;
  };
  step4: {
    generationStatus: 'pending' | 'generating' | 'completed' | 'failed' | null;
    purchaseId: string | null;
  };
  step5: {
    pacemakerWelcomed: boolean;
  };
}
```

**프론트 구현 포인트:**
```typescript
// 앱 진입 시 (또는 온보딩 페이지 진입 시)
const status = await GET('/course/onboarding/status');

if (status.isComplete) {
  router.push('/pacemaker');
} else {
  // 마지막 단계부터 재개 — 오케스트레이터가 currentStep 기반으로 렌더
}
```

### 4-1. 온보딩 5단계

**진입 조건:**
```typescript
// (app)/layout.tsx에서 초기화 후
const user = await GET('/users/me');

if (!user.hasCompletedOnboarding) {
  // v3 온보딩으로 리다이렉트
  router.replace('/onboarding');
}
```

**온보딩 오케스트레이터** (`/onboarding`)
```typescript
// 진입 시 서버 상태 확인
const { data: status } = useOnboardingStatus();

if (status?.isComplete) {
  router.replace('/pacemaker');
  return;
}

// currentStep에 따라 해당 스텝 컴포넌트 렌더
// AnimatePresence로 스텝 전환 애니메이션
```

**Step 1 — 관심 분야 선택**
```
카테고리 5개 카드 리스트
각 카드: 아이콘 + 카테고리명 + 목표 예시 문구
카드 탭 → POST /course/onboarding/step1 { category }
→ step2로 이동
```

**Step 2 — 진단 퀴즈**
```
GET /course/onboarding/step2/questions → 10문제 로드
한 문제씩 표시 + 진행률 바 (1/10 ~ 10/10)
전부 풀면 → POST /course/onboarding/step2 { answers }
→ 결과 화면 표시 (레벨 + 코스명)
→ [계속하기] → step3
```

**Step 3 — 데이터 입력**
```
상단: 기본 8개 필드 (닉네임, 나이, 은퇴나이, ...)
  → sessionStorage의 시뮬레이션 데이터로 프리필
하단: 코스별 추가 필드 (카테고리에 따라 동적 렌더링)

카테고리별 추가 필드 정의는 프론트에서 하드코딩 OK:
  '연금': [퇴직연금 유형(select), 가입 기간(number), 적립금(number)]
  '주식': [투자 경험(select), 현재 투자자산(number), 관심 유형(select)]
  '부동산': [주거 형태(select), 목표 자산(select), 청약 가점(number)]
  '세금_연말정산': [연 총소득(number), 부양가족(number), 주거형태(select), 카드사용액(number)]
  '소비_저축': [목표 저축액(number), 비상금 유무(select), 구독 수(number)]

제출 → POST /course/onboarding/step3
→ step4로 이동
```

**Step 4 — 생성 로딩**
```
POST /course/onboarding/step4/generate → 생성 시작 (멱등)
TanStack Query refetchInterval 3초 → 폴링

로딩 화면:
  단계별 메시지 애니메이션:
  [0~10초] ✅ "시은님의 데이터를 분석하고 있습니다"
  [10~30초] ⏳ "시은님만을 위한 마이북을 쓰고 있습니다"
  [30~60초] ○ "실습 미션을 준비하고 있습니다"
  진행률 바 (시간 기반 approximate)
  120초 타임아웃 → "다시 시도" 버튼

status === 'completed' → step5로 자동 이동
status === 'failed' → 에러 + [다시 시도] 버튼
```

**Step 5 — 웰컴**
```
POST /course/onboarding/step5/complete
→ welcomeMessage 표시 (페이스메이커 말풍선 스타일)
→ [홈으로 가기] → /pacemaker
→ userStore.hasCompletedOnboarding = true 즉시 설정
```

## 6. v2 → v3 마이그레이션 가이드

### v2 레거시 유저 처리

v2에서 온보딩을 완료한 유저(`onboardingVersion: 2`)는:
- 기존 기능 모두 정상 작동 (퀴즈, 페이스메이커, 서점, 마이북)
- 코스 관련 UI는 표시되지 않음 (`activeCourse: null`)
- 추후 코스 진입 경로 제공 예정 (별도 기획)

```typescript
// 코스 UI 게이팅 — onboardingVersion이 아닌 activeCourse 존재 여부로 판단
if (data.activeCourse) {
  // 코스 진도 UI 표시
}
if (data.courseBook) {
  // 코스 마이북 카드 표시
}
```

### v2 미완료 유저 처리

카카오 로그인만 하고 온보딩을 완료하지 않은 v2 유저:
- `hasCompletedOnboarding: false` → v3 온보딩으로 진입
- 기존 v2 자동 온보딩 로직 제거됨

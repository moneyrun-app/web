## 2. 서비스 비전 & 핵심 메시지

> **"부자 되세요"가 아니라 "거지 되지 마세요"**

복잡한 투자·시뮬레이션 대신, 매일 AI가 한마디씩 잔소리해주는 구조. 유저가 매일 앱을 열면 "오늘 너 얼마 쓸 수 있어", "이번 주 이거 좀 많이 썼다?", "이 글 읽어봐" 같은 메시지를 받는다.

핵심은 **습관**이다. 한 번 진단하고 끝나는 게 아니라, 매일 돌아와서 페이스메이커 메시지를 확인하는 루틴을 만드는 것.

---

## 3. 타겟 유저

20대 중후반 ~ 30대 초반 사회초년생. 돈을 벌기 시작했지만 어떻게 관리해야 하는지 모르는 사람. "투자 앱은 무섭고, 가계부는 귀찮은" 사람.

## 4. MVP 범위 — 현재 구현 완료

### 구현된 기능

| 기능 | 설명 |
|---|---|
| **시뮬레이션 랜딩** | 비로그인 7단계 시뮬레이션 (닉네임~변동비) → 재무 분석 결과 → 회원가입 유도 |
| **자동 온보딩** | 시뮬레이션 데이터 → 로그인 시 자동 프로필 생성 (별도 온보딩 페이지 불필요) |
| **페이스메이커** | AI 하루 1번 맞춤 메시지 + 일일 소비 체크 캘린더 + 월간 확정 + 퀴즈 |
| **마이북** | 3탭 (리포트 / 오답노트 / 스크랩) — V6 섹션 기반 리포트, 월간 리포트, 외부 URL 스크랩 |
| **마이페이지** | 재무 프로필 수정 → 잉여자금 자동 재계산 |
| **어드민** | 유저 관리 + 퀴즈 관리 + 운영 상수 수정 |
| **다크모드** | 전체 라이트/다크 테마 지원 |
| **반응형** | 모바일 BottomNav + 데스크탑 SideNav |

### MVP에서 제외 (추후 로드맵)

| 기능 | 제외 이유 |
|---|---|
| 커뮤니티 | 유저 수 확보 후 의미 있음 |
| 자산 트래킹 | 핵심 가치(페이스메이커)에 집중 |
| AI PDF 리포트 이메일 발송 | V6 섹션 리포트로 대체 |
| 회원 탈퇴 | UI 존재하나 API 미연결 |
| 좋은소비/고정소비 개별 CRUD | PATCH /finance/profile로 통합 처리 |

## 5. 유저 진입 & 온보딩

### 비로그인 → 시뮬레이션 → 로그인 → 자동 온보딩

```
랜딩(/) → 7단계 시뮬레이션 입력 → 결과 확인 → "카카오로 시작하기"
→ 카카오 로그인 → 시뮬레이션 데이터로 자동 프로필 생성 → /home
```

**시뮬레이션 입력 항목:**
1. 닉네임 (한글+영문, 최대 8자)
2. 현재 나이
3. 은퇴 나이
4. 연금 수령 시작 나이
5. 월 실수령액
6. 월 고정비용 (월세/보험/공과금)
7. 월 변동비용 (소비/외식)

## 6. 전체 서비스 구조

```
[비로그인]
  랜딩 (/) → 7단계 시뮬레이션 → 결과 → 로그인 유도

[로그인 후]
  ┌──────────────────────────────────────┐
  │ 하단 탭 (모바일) / 좌측 사이드 (데스크탑) │
  │ 페이스메이커 | 마이북 | 마이           │
  └──────────────────────────────────────┘
  + 어드민 (admin 역할만)
```

| 탭 | 라우트 | 핵심 역할 |
|---|---|---|
| **페이스메이커** | `/home` | AI 메시지 + 소비 트래킹 + 월간 확정 + 퀴즈 |
| **마이북** | `/book` | 리포트 / 오답노트 / 스크랩 |
| **마이** | `/my` | 재무 프로필 수정 |
| **어드민** | `/admin/*` | 유저·퀴즈·상수 관리 (admin만) |

## 7. 잉여자금 자동 계산 시스템

### 핵심 공식

```
잉여자금(월) = monthlyIncome - monthlyFixedCost - monthlyVariableCost
잉여자금(주) = 잉여자금(월) / 4.3
잉여자금(일) = 잉여자금(월) / 30
```

(현재 구현은 `monthlyIncome - fixedCost - variableCost` 방식)

### 등급 판정

```
변동비 비율 = (fixedCost + variableCost) / monthlyIncome
RED    : 비율 >= 70%
YELLOW : 50% <= 비율 < 70%
GREEN  : 비율 < 50%
```

### 재계산 트리거

마이페이지에서 PATCH /finance/profile → 즉시 전체 재계산.

## 8. 페이스메이커 (홈)

AI가 먼저 말을 거는 구조 + 유저가 매일 소비를 기록하는 습관.

### 표시 정보

1. **오늘의 AI 메시지** (테마 태그 + 한마디)
2. **월간 소비 캘린더** (일별 초록/노랑/빨강 상태)
3. **데일리 퀴즈** (객관식 4지선다 + 해설)
4. **월간 확정** (월말 소비 데이터 잠금)
5. 하루/주/월 예산 표시

### 메시지 톤 가이드

- **찐친이 잔소리** — 걱정하면서도 직접적으로 말하는 톤
- 등급별 톤 차이: RED(직접적+유머), YELLOW(은근히 찌르기), GREEN(격려)

---

## 9. 마이북

내 금융 관련 콘텐츠가 모이는 공간. 3탭 구조.

### 탭 구조 (현행)

| 탭 | 설명 |
|---|---|
| **리포트** | AI 시뮬레이터 분석 리포트 (V6 섹션 기반 26개 컴포넌트) + 월간 리포트 (체감+제안+메모 기반 생성) |
| **오답노트** | 틀린 퀴즈 (내 답 빨강 + 정답 초록 + 간단/상세 설명) |
| **스크랩** | 외부 URL 스크랩 (AI 요약 포함) |

---

## 10. 마이페이지

1. **재무 프로필 수정** — 닉네임, 나이, 은퇴나이, 연금나이, 실수령, 고정비, 변동비
2. **일/주/월 예산 + 잉여자금 표시** — 수정 즉시 재계산
3. **계정 설정** — 마케팅 동의 (미구현), 회원 탈퇴 (미구현)

## 11. 어드민 시스템

### 어드민 페이지 (admin 역할만 접근)

| 페이지 | 기능 |
|---|---|
| `/admin/users` | 유저 목록 (페이지네이션), 역할/온보딩 상태 확인 |
| `/admin/quizzes` | 퀴즈 목록 (펼침/접기), 문제/정답/해설 확인 |
| `/admin/config` | 운영 상수 실시간 수정 |

### system_config 관리 항목

| key | 기본값 | 설명 |
|---|---|---|
| `seoul_avg_rent` | 730,000 | 서울 평균 월세 |
| `avg_food` | 420,000 | 서울 2030 평균 식비 |
| `avg_transport` | 80,000 | 교통 평균 |
| `avg_subscription` | 50,000 | 구독 평균 |
| `avg_shopping` | 150,000 | 쇼핑 평균 |
| `avg_leisure` | 220,000 | 여가 평균 |
| `avg_etc` | 100,000 | 기타 평균 |
| `inflation_rate` | 0.025 | 인플레이션율 |

## 12. 데이터 모델

| `user_content_reads` | 읽음 기록 |
| `user_content_scraps` | 학습 콘텐츠 스크랩 기록 |
| `system_config` | 운영 상수 |
| `quizzes` | 퀴즈 풀 (문제, 선택지, 정답, 해설, 카테고리) |

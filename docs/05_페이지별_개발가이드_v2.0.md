# 머니런 페이지별 개발가이드

> **버전:** v2.0 — 3페이지 MVP (페이스메이커/마이북/마이페이지)

---

## 0. 온보딩 `/onboarding`

```
라우트:    /onboarding
인증:      필요 (로그인 직후, 최초 1회)
```

**목적:** 나이 + 월 실수령액 + 좋은 소비 + 고정 소비 입력 → "하루 OO원 쓸 수 있어" 결과.

**스텝 구성:**

| 스텝 | 내용 | 컴포넌트 |
|---|---|---|
| 1 | 나이 입력 | `AgeInput` |
| 2 | 월 실수령액 입력 | `IncomeInput` |
| 3 | 좋은 소비 입력 (적금/투자/연금/IRP/보험) | `GoodSpendingInput` |
| 4 | 고정 소비 입력 (월세/관리비/통신비) | `FixedExpenseInput` |
| 결과 | 잉여자금 표시 → "하루 OO원 쓸 수 있어" | `SurplusResult` |

**API:** 마지막에 `POST /onboarding` 1회 호출.

**주의:**
- 스텝 전환: 슬라이드 애니메이션
- 좋은 소비: 항목 추가/삭제 가능, 0원 가능
- 고정 소비: 월세 0원 가능 (부모님 집)
- 월세 입력 시 "서울 평균 73만 원" 참고 표시
- 결과 화면에서 등급 뱃지 + "하루 OO원" 크게 표시 → "시작하기" 버튼 → `/home`

## 1. 페이스메이커 (홈) `/home`

```
라우트:    /home
인증:      필요
```

**목적:** 매일 열어보는 메인 화면. AI가 오늘의 한마디를 보여준다.

**화면 구성:**
1. 등급 뱃지 + "하루 OO원 쓸 수 있어" (상단)
2. **AI 메시지 카드** (핵심, 크게)
3. 추천 행동 1~2개 (마이북 콘텐츠 링크)

**컴포넌트:** `pacemaker/TodayMessage`, `DailySurplus`, `ActionCard`

**상태:**
```typescript
const { data } = useQuery({ queryKey: ['pacemaker-today'], queryFn: () => api.get('/pacemaker/today') });
const { surplus, grade } = useFinanceStore();
```

**API:** `GET /pacemaker/today` — 오늘 메시지 없으면 AI 생성 후 반환.

**주의:**
- 메시지 로딩 중: "오늘의 한마디 준비 중..." 스켈레톤
- 추천 행동 클릭 → 마이북 해당 콘텐츠로 이동
- 같은 날 다시 열어도 같은 메시지 (백엔드 캐시)

## 2. 마이북 `/book`

```
라우트:    /book (목록), /book/:id (상세)
인증:      필요
```

**목적:** 내 금융 콘텐츠 모음. 카테고리별, 날짜별.

**카테고리 탭:**
```
[상세 리포트] [주간 리포트] [스크랩] [금융 학습]
```

**컴포넌트:** `book/CategoryTabs`, `ContentListItem`, `ContentDetail`, `ReportDetail`, `ScrapButton`

**탭별 API:**

| 탭 | API |
|---|---|
| 상세 리포트 | `GET /book/detailed-reports` (canGenerate 포함) |
| 주간 리포트 | `GET /book/weekly-reports`, `POST /book/weekly-reports` |
| 스크랩 | `GET /book/scraps` |
| 금융 학습 | `GET /book/learn` |

**주의:**
- 날짜별 정렬 (최신순)
- 상세 리포트: 월 1회 제한. `canGenerate` false면 "OO일 후 가능" 안내
- 주간 리포트: "이번 주 리포트 만들기" 버튼 → 유저 입력 → POST → AI 가이드 생성
- 학습 콘텐츠: 읽음 표시 (조회 시 자동)
- 학습 콘텐츠 상세: react-markdown + @tailwindcss/typography

## 3. 마이페이지 `/my`

```
라우트:    /my
인증:      필요
```

**목적:** 재무 정보 수정 (→ 잉여자금 재계산) + 계정 설정.

**화면 구성:**
1. 닉네임 + 등급 뱃지
2. 재무 정보 수정 (나이, 실수령액, 좋은 소비, 고정 소비)
3. 계정 설정 (알림, 마케팅 동의)
4. 회원 탈퇴

**컴포넌트:** `my/ProfileEditForm`, `GoodSpendingEditor`, `FixedExpenseEditor`, `AccountSettings`

**API:**

| API | 시점 |
|---|---|
| `GET /finance/profile` | 진입 시 |
| `PATCH /finance/profile` | 나이/실수령액 수정 |
| `POST/PATCH/DELETE /finance/good-spendings` | 좋은 소비 CRUD |
| `PATCH /finance/fixed-expenses` | 고정 소비 수정 |
| `PATCH /users/me` | 이메일/마케팅 동의 수정 |
| `DELETE /users/me` | 회원 탈퇴 |

**주의:**
- 수정 → Zustand 즉시 업데이트 + API 저장
- 잉여자금 즉시 재계산 → 홈 탭 반영
- 회원 탈퇴: 확인 다이얼로그 필수

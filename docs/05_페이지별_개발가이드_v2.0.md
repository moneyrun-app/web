# 머니런 페이지별 개발가이드

> **버전:** v3.0 — 현행 코드 기준 전면 업데이트
> **최종 수정:** 2026.04.08

---

## 전체 라우트 맵

```
/                           ← 랜딩 (시뮬레이션 위저드)     [public]
/(app)/home                 ← 페이스메이커 (홈)            [인증 필요]
/(app)/book                 ← 마이북                       [인증 필요]
/(app)/book/[id]?type=      ← 리포트 상세                  [인증 필요]
/(app)/my                   ← 마이페이지                   [인증 필요]
/(app)/admin/users          ← 어드민 유저                  [어드민 전용]
/(app)/admin/quizzes        ← 어드민 퀴즈                  [어드민 전용]
/(app)/admin/config         ← 어드민 설정                  [어드민 전용]
/api/auth/[...nextauth]     ← NextAuth 핸들러
/api/proxy/[...path]        ← CORS 프록시
```

## 0. 랜딩 페이지 `/` (public)

```
라우트:    /
인증:      불필요 (비로그인)
레이아웃:  (public)/layout.tsx — 헤더(로고 + 테마토글 + 로그인 버튼)
```

**목적:** 7단계 시뮬레이션 위저드로 유저의 재무 상태를 체험시키고, 회원가입 유도.

**7단계 시뮬레이션:**

| 스텝 | 내용 | 검증 |
|---|---|---|
| 0 | 히어로 ("65세에 거지 되지 마세요") | 진입점, 클릭 시 시작 |
| 1 | 닉네임 입력 | 한글+영문, 최대 8자 |
| 2 | 현재 나이 | 숫자 |
| 3 | 은퇴 나이 | 숫자 |
| 4 | 연금 수령 시작 나이 | 숫자 |
| 5 | 월 실수령액 | 만원 단위 표시 |
| 6 | 월 고정비용 (월세/보험/공과금) | 만원 단위 표시 |
| 7 | 월 변동비용 (소비/외식) | 만원 단위 표시 |

**컴포넌트:**
- `SimulationResult` — 결과 3단계 캐러셀
  - `StepCurrentReport` — 현재 재무 등급 + 잉여자금 + 지출 구조
  - `StepFutureProjection` — 투자 시나리오 (3%/7%/10% 수익률)
  - `StepActionCTA` — 회원가입 유도 CTA
- `InvestmentChart` — 투자 수익 차트
- `TrajectoryChart` — 자산 궤적 차트

**상태:** `useSimulationStore` (sessionStorage 영속화)

**API:** 없음 — 프론트 자체 계산 (`lib/simulation.ts`)

**특이사항:**
- Framer Motion 슬라이드 애니메이션 (좌→우 전환)
- Enter/ESC 키보드 네비게이션
- 금액 입력 시 만원 접미사 오버레이
- 소득 변경 시 비용이 소득 초과하면 자동 스케일링
- 세션 스토리지에 입력값 저장 → 로그인 후 온보딩에 활용

## 1. 앱 레이아웃 `/(app)/layout.tsx`

```
라우트:    /(app)/* 래퍼
인증:      필요 — 미인증 시 / 로 리다이렉트
```

**목적:** 인증 가드 + 초기화 + 네비게이션 제공

**초기화 플로우:**
```
1. sessionStorage에서 JWT 복원 시도
2. 없으면 NextAuth 카카오 토큰 → POST /auth/kakao → JWT 획득
3. isNewUser면:
   - 세션 스토리지의 시뮬레이션 데이터로 POST /auth/onboarding
   - 시뮬레이션 데이터 삭제
4. 기존 유저면:
   - GET /finance/profile → financeStore 동기화
   - GET /users/me → userStore 동기화
5. 시뮬레이션 데이터와 DB 프로필이 다르면 → 업데이트 다이얼로그
```

**컴포넌트:**
- `SideNav` — 데스크탑 사이드바 (홈/마이북/마이/어드민)
- `BottomNav` — 모바일 하단 탭
- `ThemeToggle` — 다크모드 토글
- `GradeProvider` — 등급별 CSS 변수 주입
- `ConfirmDialog` — 프로필 충돌 시 업데이트 확인

**API:**
- `POST /auth/kakao`
- `POST /auth/onboarding`
- `GET /finance/profile`
- `GET /users/me`

## 2. 페이스메이커 (홈) `/home`

```
라우트:    /home
인증:      필요
```

**목적:** 매일 열어보는 메인 화면. AI 메시지 + 소비 트래킹 + 퀴즈.

**화면 구성:**

### 2-1. 페이스메이커 메시지 (상단)
- AI 생성 오늘의 한마디
- 테마 태그 + 영감 문구

### 2-2. 월간 소비 트랙 (캘린더)
- 7일 단위 행 × 해당 월 주 수
- 일별 색상: 초록(절약) / 노랑(보통) / 빨강(초과)
- 호버 시 금액+상태 툴팁
- 좌우 스와이프로 월 이동
- 일 클릭 → 데일리 체크 모달 (상태+금액 입력)

### 2-3. 월간 확정 컨트롤
- 확정 버튼 → 해당 월 데이터 잠금
- 취소 버튼 → 잠금 해제
- 미확정 과거 월 자동 알림

### 2-4. 데일리 퀴즈
- 객관식 4지선다
- 정답 제출 후 설명 표시 (간단 + 상세)
- 마크다운 렌더링
- 진행률 (X / N 퀴즈)

**컴포넌트:**
- `TrafficLight` — 일일 예산 잔여 비율 표시
- `InvestmentTier` — 투자 등급 + 일/주/월 예산
- `GradeBadge` — 등급 뱃지
- `Markdown` — 퀴즈 설명 렌더링

**API:**
```
GET  /pacemaker/today                    ← 오늘 메시지 + 퀴즈
POST /pacemaker/quiz/:quizId/answer      ← 퀴즈 답변
POST /pacemaker/feedback                 ← 메시지 피드백
GET  /pacemaker/daily-checks?month=      ← 월별 체크 조회
GET  /pacemaker/weekly-summary?date=     ← 주간 요약
POST /pacemaker/daily-check              ← 일일 체크 제출
GET  /pacemaker/monthly-finalize-status  ← 확정 상태
POST /pacemaker/monthly-finalize         ← 월간 확정
POST /pacemaker/monthly-finalize/cancel  ← 확정 취소
```

**상태:** `useFinanceStore` (variableCost: 일/주/월 예산)

## 3. 마이북 `/book`

```
라우트:    /book
인증:      필요
```

**목적:** 내 금융 콘텐츠 라이브러리.

**3개 탭:**

| 탭 | 내용 |
|---|---|
| 리포트 | 시뮬레이터 분석 리포트 + 월간 리포트 (구분선으로 분리) |
| 오답노트 | 틀린 퀴즈 (내 답 빨강 + 정답 초록 + 설명) |
| 스크랩 | 외부 URL 스크랩 (AI 요약 포함) |

### 3-1. 리포트 탭
- 상세 리포트: 날짜별 카드 → 클릭 시 `/book/{id}?type=detailed`
- 월간 리포트: 대기(pending) / 생성됨 상태 구분
- 월간 리포트 생성 모달 (3단계: 체감 → 제안 체크 → 메모)

### 3-2. 오답노트 탭
- 펼침/접기 카드
- 내 답 (빨강) vs 정답 (초록) 배지
- 한줄 설명 + 펼침 시 상세 설명 (마크다운)

### 3-3. 스크랩 탭
- 그리드 레이아웃 (모바일 1열, 데스크탑 2열)
- 채널명, 작성자, AI 요약, 원본 링크
- FAB 버튼 → URL 입력 모달 → POST /book/scraps

**컴포넌트:**
- `CategoryTabs` — 3탭 전환
- `MonthlyReportCreate` — 월간 리포트 생성 모달
- `Markdown` — 설명 렌더링

**API:**
```
GET  /book/detailed-reports     ← 리포트 목록
GET  /book/monthly-reports      ← 월간 리포트 목록
GET  /book/wrong-notes          ← 오답노트
GET  /book/scraps               ← 스크랩 목록
POST /book/scraps               ← 스크랩 추가
```

## 4. 리포트 상세 `/book/[id]`

```
라우트:    /book/[id]?type=detailed|monthly
인증:      필요
```

**목적:** 상세 리포트 / 월간 리포트 뷰어.

**타입 판별:** URL 쿼리 파라미터 `?type=`

### 4-1. 상세 리포트 (V6)
- `reportVersion === 'v6'` + `sections` 배열 존재 시
- `V6ReportDetail` 컴포넌트로 26개 섹션 타입 동적 렌더링
- 섹션 타입: hero_card, markdown, gauge_chart, donut_chart, comparison_card, stacked_bar, saving_opportunity, bar_chart, progress_card, simulation_table, tip_card, action_checklist, line_chart, comparison_highlight, compound_effect, world_comparison, age_roadmap, macro_indicators, inflation_impact, goal_tracker, tax_benefit, investment_pyramid, portfolio_suggestion, disclaimer 등

### 4-2. 월간 리포트 (V2)
- `MonthlyReportV2Detail` 컴포넌트
- 소비 분석, 카테고리 분석 등 섹션 기반 렌더링

### 4-3. 레거시 폴백
- sections 없는 경우 마크다운 문자열 수동 파싱
- `##`, `###`, `-`, `<br>` 처리

**API:**
```
GET /book/detailed-reports/:id   ← type=detailed
GET /book/monthly-reports/:id    ← type=monthly
```

## 5. 마이페이지 `/my`

```
라우트:    /my
인증:      필요
```

**목적:** 재무 프로필 수정 → 잉여자금 재계산.

**화면 구성:**

### 5-1. 유저 헤더
- 아바타 + 닉네임 + 등급 뱃지
- 프로필 오래된 경우 stale 경고

### 5-2. 재무 프로필 카드
- 일/주/월 예산 표시
- 잉여자금 (양수: 초록, 음수: 빨강)
- 편집 가능 필드:
  - 닉네임 (텍스트, 최대 8자)
  - 나이, 은퇴 나이, 연금 수령 나이 (숫자)
  - 월 실수령, 고정비, 변동비 (만원 단위)
- 실시간 계산: 총지출, 잉여자금, 투자 기간, 수령 기간
- 저장 버튼 (로딩 상태)

### 5-3. 계정 설정 카드
- 마케팅 동의 토글 (미구현, coming soon)
- 회원 탈퇴 버튼 (미구현)

**컴포넌트:**
- `GradeBadge` — 등급 표시
- `EditRow` — 포맷 숫자 입력 커스텀 컴포넌트

**API:**
```
GET   /finance/profile           ← 프로필 로드
PATCH /finance/profile           ← 전체 프로필 저장
```

**상태:** `useUserStore`, `useFinanceStore` + 로컬 편집 필드들

## 6. 어드민 페이지 `/admin/*`

### 6-1. 유저 관리 `/admin/users`

- 유저 수 요약 카드
- 페이지네이션 테이블 (20건/페이지)
- 컬럼: 닉네임, 이메일, 역할 (admin 빨강 배지), 온보딩 상태 (색상 점), 가입일

**API:** `GET /admin/users?page=&limit=`

### 6-2. 퀴즈 관리 `/admin/quizzes`

- 퀴즈 수 요약 카드
- 펼침/접기 퀴즈 카드
  - 카테고리 배지 + 문제
  - 4지선다 (정답 초록 하이라이트)
  - 간단 설명 + 상세 설명 (마크다운)
  - 출처, 생성일

**API:** `GET /admin/quizzes`

### 6-3. 설정 관리 `/admin/config`

- 운영 상수 편집 (서울 평균 월세, 카테고리 평균, 인플레이션율 등)
- API 양방향 싱크

**API:** `PATCH /admin/constants/:key`

## 레이아웃 구조

```
app/layout.tsx                    ← 루트: SessionProvider + QueryProvider + ThemeInit
├── (public)/layout.tsx           ← 비로그인: 헤더(로고 + 테마 + 로그인)
│   └── page.tsx                  ← 랜딩/시뮬레이션
└── (app)/layout.tsx              ← 인증: SideNav(데스크탑) + BottomNav(모바일) + 인증가드
    ├── home/page.tsx
    ├── book/page.tsx
    ├── book/[id]/page.tsx
    ├── my/page.tsx
    └── admin/
        ├── users/page.tsx
        ├── quizzes/page.tsx
        └── config/page.tsx
```

**반응형:**
- 모바일: BottomNav (하단 탭) + 모바일 헤더 (테마 토글)
- 데스크탑: SideNav (좌측 사이드바, md:pl-60)

## 1. 완료된 프론트엔드 변경사항

### 1-1. 마이북 탭 구조 변경

| 변경 전 | 변경 후 |
|---------|---------|
| AI리포트 / 월간 / 오답노트 / 스크랩 / 학습 (5탭) | 리포트 / 오답노트 / 스크랩 (3탭) |

- **"월간" 탭 제거** → 리포트 탭 안에 구분선 아래로 합침
- **"학습" 탭 제거** → 기능 삭제
- **"AI리포트" → "리포트"** 라벨 변경

### 1-2. 리포트 탭 레이아웃


### 1-4. 오답노트 개편

**변경 전:**
- 정답 표시 + 다시풀기 버튼 + O/X 재도전 + 복습완료 섹션

**변경 후:**
- 내 답 (빨강 배지) + 정답 (초록 배지) + 한줄 설명
- "자세히 보기" 누르면 상세 설명 펼침/접기
- 다시풀기 기능 삭제
- 복습완료 섹션 삭제

### 1-5. 라우팅 방식 변경

상세 페이지 타입 판별을 `id` prefix (`dr-`, `mr-`) 에서 **쿼리 파라미터** 방식으로 변경.

```
/book/{uuid}?type=detailed  → 시뮬레이터 분석 리포트
/book/{uuid}?type=monthly   → 월간 리포트
```

## 2. 백엔드 요청사항 (미완료)

### 2-1. 삭제할 엔드포인트

| 메서드 | 경로 | 이유 |
|--------|------|------|
| GET | `/book/detailed-reports/:id/pdf` | PDF 다운로드 삭제 |

| GET | `/book/learn/:id` | 학습 기능 삭제 |
| POST | `/book/learn/:id/scrap` | 학습 기능 삭제 |

### 2-2. 변경할 엔드포인트

#### `GET /book/detailed-reports` — 목록

```json
{
  "items": [{
    "id": "uuid",
    "title": "시뮬레이터 분석 리포트",
    "summary": "변동비 1,500,000원, 하루 50,000원 사용 가능",
    "analyzedAt": "2026-04-05T00:00:00.000Z",
    "createdAt": "2026-04-05T00:00:00.000Z"
  }]
}
```
- `pdfUrl` 필드 제거
- `analyzedAt` 필드 추가
- `title`: "시뮬레이터 분석 리포트"
- `summary`: 시뮬레이션 결과 기반 한줄 요약

#### `GET /book/detailed-reports/:id` — 상세

```json
{
  "id": "uuid",
  "title": "시뮬레이터 분석 리포트",
  "content": "{ \"sections\": [...] }",
  "analyzedAt": "2026-04-05T00:00:00.000Z",
  "createdAt": "2026-04-05T00:00:00.000Z"
}
```
- `content`는 JSON 문자열 (프론트에서 `JSON.parse` 처리함)
- `pdfUrl` 제거
- `content.sections` 배열 구조는 위 섹션 타입 참고

#### `GET /book/wrong-notes` — 오답노트

```json
[{
  "id": "uuid",
  "quizId": "uuid",
  "question": "코스피와 코스닥은 같은 시장이다",
  "userAnswer": true,
  "correctAnswer": false,
  "explanation": "코스피와 코스닥은 별개의 시장입니다",
  "detailedExplanation": "코스피(KOSPI)는 유가증권시장에 상장된 대형 기업들의...",
  "source": "시뮬레이터 분석 리포트",
  "category": "주식",
  "createdAt": "2026-04-05T00:00:00.000Z"
}]
```
- `userAnswer` (boolean) 추가: 유저가 선택한 답
- `detailedExplanation` (string) 추가: AI 생성 상세 설명
- `isResolved` 필드 제거

### 2-3. 새로 구현할 로직

#### 틀린 문제 퀴즈 재출제
- `GET /pacemaker/today` 퀴즈 생성 시 `wrong_notes` 테이블에서 일정 확률(30%)로 재출제
- 재출제 퀴즈의 `source`를 `"오답노트 복습"`으로 표시
- 재출제에서 맞추면 `wrong_notes`에서 해당 레코드 삭제

#### 오답 상세 설명 자동 생성
- 퀴즈 오답 발생 시 AI가 `detailedExplanation` 생성
- `wrong_notes` 테이블에 `detailed_explanation TEXT` 컬럼 추가
- `user_answer BOOLEAN` 컬럼 추가

### 2-4. DB 마이그레이션

```sql
-- wrong_notes 테이블
ALTER TABLE wrong_notes ADD COLUMN user_answer BOOLEAN;
ALTER TABLE wrong_notes ADD COLUMN detailed_explanation TEXT;
ALTER TABLE wrong_notes DROP COLUMN is_resolved;  -- 또는 유지하되 사용 안함
```

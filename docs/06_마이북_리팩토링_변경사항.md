## 1. 완료된 프론트엔드 변경사항

### 1-1. 마이북 탭 구조 변경

| 변경 전 | 변경 후 |
|---------|---------|
| AI리포트 / 월간 / 오답노트 / 스크랩 / 학습 (5탭) | 리포트 / 오답노트 / 스크랩 (3탭) |

- **"월간" 탭 제거** → 리포트 탭 안에 구분선 아래로 합침
- **"학습" 탭 제거** → 기능 삭제
- **"AI리포트" → "리포트"** 라벨 변경

### 1-2. 리포트 탭 레이아웃

```
┌────────────────────────────────┐
│ 2026.04.05 분석                │
│ 시뮬레이터 분석 리포트         │
│ 요약 텍스트...        [상세보기]│
└────────────────────────────────┘
────────── 구분선 ──────────
  월간 리포트
┌────────────────────────────────┐
│ 2026-03 월간 리포트   [대기중]  │  ← pending 상태
│ 2026-02 월간 리포트   [보기]    │  ← created 상태
└────────────────────────────────┘
```

- 시뮬레이터 분석 리포트: **풀너비 카드**, 날짜 `analyzedAt` 기반
- summary 길면 `...` 처리 + 상세보기 버튼 같은 줄 오른쪽
- PDF 다운로드 버튼 **전면 삭제** (목록 + 상세 모두)
- 스크랩 FAB 버튼 색상: 노란색 → **검정색**
- 월간 리포트 생성: 3단계 모달 (체감 → 제안 체크 → 메모)

### 1-4. 오답노트 개편

**변경 전:**
- 정답 표시 + 다시풀기 버튼 + O/X 재도전 + 복습완료 섹션

**변경 후:**
- 내 답 (빨강 배지) + 정답 (초록 배지) + 한줄 설명
- "자세히 보기" 누르면 상세 설명 펼침/접기 (마크다운)
- 다시풀기 기능 삭제
- 복습완료 섹션 삭제

### 1-5. 라우팅 방식 변경

상세 페이지 타입 판별을 `id` prefix (`dr-`, `mr-`) 에서 **쿼리 파라미터** 방식으로 변경.

```
/book/{uuid}?type=detailed  → 시뮬레이터 분석 리포트
/book/{uuid}?type=monthly   → 월간 리포트
```

## 2. 백엔드 요청사항

### 2-1. 삭제 가능한 엔드포인트 (프론트 미사용)

| 메서드 | 경로 | 이유 |
|--------|------|------|
| GET | `/book/detailed-reports/:id/pdf` | PDF 다운로드 삭제 |
| POST | `/book/wrong-notes/:id/retry` | 다시풀기 삭제 |
| PATCH | `/users/me` | 프론트 미사용 (프로필은 /finance/profile로 처리) |
| DELETE | `/users/me` | 회원 탈퇴 미구현 (추후 P1) |
| POST | `/finance/good-spendings` | 프론트 미사용 (온보딩에서 통합 처리) |
| PATCH | `/finance/good-spendings/:id` | 프론트 미사용 |
| DELETE | `/finance/good-spendings/:id` | 프론트 미사용 |
| PATCH | `/finance/fixed-expenses` | 프론트 미사용 |
| GET | `/pacemaker/history` | 히스토리 페이지 없음 |
| GET | `/book/weekly-reports` | 주간 → 월간 전환 |
| POST | `/book/weekly-reports` | 주간 → 월간 전환 |
| GET | `/book/weekly-reports/:id` | 주간 → 월간 전환 |

> **참고:** PATCH/DELETE `/users/me`, `/finance/good-spendings`, `/finance/fixed-expenses`는 P1에서 마이페이지 개별 CRUD 구현 시 필요할 수 있으므로 비활성화 수준 권장. 나머지(PDF, retry, 주간 리포트)는 확실히 삭제 가능.

### 2-2. 변경할 엔드포인트

#### `GET /book/detailed-reports` — 목록

```json
{
  "items": [{
    "id": "uuid",
    "title": "시뮬레이터 분석 리포트",
    "summary": "변동비 1,500,000원, 하루 50,000원 사용 가능",
    "analyzedAt": "2026-04-05T00:00:00.000Z",
    "createdAt": "2026-04-05T00:00:00.000Z"
  }]
}
```
- `pdfUrl` 필드 제거
- `analyzedAt` 필드 추가

#### `GET /book/wrong-notes` — 오답노트

```json
[{
  "id": "uuid",
  "quizId": "uuid",
  "question": "코스피와 코스닥은 같은 시장이다",
  "userAnswer": 2,
  "correctAnswer": 3,
  "explanation": "코스피와 코스닥은 별개의 시장입니다",
  "detailedExplanation": "코스피(KOSPI)는 유가증권시장에 상장된...",
  "source": "시뮬레이터 분석 리포트",
  "category": "주식",
  "createdAt": "2026-04-05T00:00:00.000Z"
}]
```
- `userAnswer` (number) 추가: 유저가 선택한 답 인덱스
- `detailedExplanation` (string) 추가: AI 생성 상세 설명

### 2-3. 새로 구현할 로직

#### 틀린 문제 퀴즈 재출제
- `GET /pacemaker/today` 퀴즈 생성 시 `wrong_notes` 테이블에서 일정 확률(30%)로 재출제
- 재출제에서 맞추면 `wrong_notes`에서 해당 레코드 삭제

#### 오답 상세 설명 자동 생성
- 퀴즈 오답 발생 시 AI가 `detailedExplanation` 생성
- `wrong_notes` 테이블에 `detailed_explanation TEXT` 컬럼 추가
- `user_answer INTEGER` 컬럼 추가

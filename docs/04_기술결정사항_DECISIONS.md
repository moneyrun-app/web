**영향 범위:** 전 문서 (01~06 + CLAUDE.md + PROGRESS.md)

## [2026-04-07] 월간 리포트 v2 + 홈 인풋 커서 개선

**결정:** 주간 리포트 → 월간 리포트로 전환, 3단계 생성 (체감→제안→메모)

**이유:**
- 주간 리포트는 빈도가 너무 잦아 유저 피로감
- 월간 단위가 소비 패턴 분석에 더 의미 있음

**API 추가:**
- `GET /book/monthly-reports`
- `GET /book/monthly-reports/:id`
- `GET /book/monthly-reports/proposals`
- `POST /book/monthly-reports`

**제거:** 주간 리포트 API (`GET/POST /book/weekly-reports`) 프론트 미사용

## [2026-04-06] 시뮬레이션 랜딩 페이지

**결정:** 비로그인 랜딩을 7단계 시뮬레이션 위저드로 전환

**이유:**
- 단순 로그인 버튼만 있는 랜딩 → 전환율 낮음
- 시뮬레이션 체험 후 로그인하면 데이터 자동 온보딩 → 마찰 최소화

**결과:** 별도 온보딩 페이지 불필요, 시뮬레이션 → 로그인 → 자동 프로필 생성

## [2026-04-02] MVP v2.0 — 3페이지로 대폭 축소

**결정:** 5탭 풀 기능 → 3페이지(페이스메이커/마이북/마이페이지) 집중 MVP

**이유:**
- MVP는 핵심 가치 검증이 목적. 페이스메이커(AI 일일 코칭) + 마이북(콘텐츠 모음)이 핵심
- 빠른 출시 > 완벽한 기능

**제거된 기능:** 커뮤니티, 자산 트래킹, 65세 시뮬레이션 독립 페이지, AI PDF 이메일 발송, 주간 체크인 UI, 소득 구간

## [2026-04-02] 온보딩 간소화 — 7문항 → 4스텝

**결정:** 진단 7문항을 4스텝 온보딩(나이, 실수령액, 좋은 소비, 고정 소비)으로 축소

**이유:**
- 핵심은 "잉여자금 = 하루 쓸 수 있는 돈" 계산에 필요한 최소 데이터
- 비로그인 진단 제거 → 로그인 후 바로 온보딩

**현재 상태:** 온보딩 페이지 자체가 사라지고, 시뮬레이션 데이터로 자동 온보딩
